# Accessibility Statement for Risky Weather

We are committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards.

## Conformance Status

The Web Content Accessibility Guidelines (WCAG) defines requirements for designers and developers to improve accessibility for people with disabilities. It defines three levels of conformance: Level A, Level AA, and Level AAA.

Risky Weather is partially conformant with **WCAG 2.2 level AA**. Partially conformant means that some parts of the content do not fully conform to the accessibility standard.

## Measures to Support Accessibility

We have taken the following measures to ensure accessibility:
*   **Semantic HTML**: Using proper heading structures (`h1`-`h3`), landmarks (`header`, `main`, `section`), and lists.
*   **Color Contrast**: Ensuring text meets the 4.5:1 contrast ratio requirement against its background.
*   **Keyboard Navigation**: Ensuring all interactive elements are focusable and have visible focus indicators.
*   **Screen Reader Support**: Using ARIA labels where visual labels are hidden (e.g., search inputs, toggle buttons).
*   **Responsive Design**: Ensuring the application is usable on various device sizes and zoom levels.

## Compatibility with Browsers and Assistive Technology

Risky Weather is designed to be compatible with the following assistive technologies:
*   Modern web browsers (Chrome, Firefox, Safari, Edge)
*   Screen readers (VoiceOver, NVDA, JAWS)

## Technical Specifications

Accessibility of Risky Weather relies on the following technologies to work with the particular combination of web browser and any assistive technologies or plugins installed on your computer:
*   HTML
*   WAI-ARIA
*   CSS
*   JavaScript
